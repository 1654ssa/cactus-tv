# CactusStreamflow v0.8.6

CactusStreamflow 使用 Cloudflare Cache API 为 HLS 视频建立前方约 600 秒的边缘缓存。它不需要 R2、Queue、信用卡或额外 Worker。

## v0.8.6 的安全边界

CactusStreamflow 只处理已经明确加入数据源“媒体域名白名单”的 HTTPS 地址。

未知动态 CDN 不再被强制送入 Cloudflare 代理，而是保持浏览器直连。原因是部分影视 CDN 会把临时播放地址绑定到访问者 IP、浏览器请求头、Cookie 或 Referer。浏览器能够播放，不代表 Cloudflare 服务器也能代替用户拉取。

v0.8.4 的临时媒体凭证方案会强制代理这些动态 CDN，导致部分视频完全打不开，因此已在 v0.8.6 撤销。

## 播放优先级

1. 媒体域名在白名单中且数据源开启代理：优先通过 `/api/stream` 播放，并允许 Streamflow 预取。
2. 代理播放失败：播放器自动继续尝试同一集的原始直连地址。
3. 媒体域名不在白名单：直接使用原始地址播放，不启动 Streamflow。
4. 关闭 Streamflow：只停止边缘预取，不影响普通代理或直连播放。

## 600 秒滚动窗口

识别到 HLS 并确认首帧正常后，系统分批缓存当前位置之后最多 600 秒的分片：

```text
当前位置 → Cloudflare 边缘节点预取未来分片 → 播放器随后从边缘缓存读取
```

播放时每批最多处理 4 个对象，暂停时最多处理 6 个对象。任务之间有锁，避免重复预取与正常播放争抢带宽。

## 两种缓冲不是一回事

播放器诊断面板中的“缓存 30s”表示设备本地缓冲。Streamflow 面板中的“300 / 600 秒”表示 Cloudflare 边缘节点已准备的内容。边缘缓存不会提前显示在浏览器原生进度条中。

## 面板

播放器中的 Streamflow 面板会显示：

- 当前是否运行；
- 已覆盖秒数；
- 新增与复用分片；
- 实际播放 HIT/MISS；
- 无法启动的具体原因。

面板可以关闭，关闭后保留轻量状态胶囊。

## 部署

已有 Cactus TV 用户升级 v0.8.6 只需同步代码并等待 Pages 自动部署。Cloudflare 后台、D1、Bindings 均无需修改。
